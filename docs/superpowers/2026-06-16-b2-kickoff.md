## b2 design essentials (from the plan/spec)
- **Tick-based state machine** (off the poll or a self-cancelling `set_interval`), NEVER a
  blocking UI-thread wait (a blocking wait freezes every pane — reader threads block on
  `call_from_thread`). States: inject `/handoff` -> await idle -> extract the
  `NEW SESSION PROMPT` block from the transcript's last assistant message -> **confirm
  modal** (show the prompt; Enter=proceed/Esc=cancel) -> inject `/clear` -> detect child
  sid -> inject reseed prompt -> `_set_lineage(child, parent, parent_jsonl)`.
- **Human-gated**: saikai NEVER types `/clear` autonomously — only after the user confirms
  a *shown* handoff. Anchor on the standard **`/handoff` skill** (not a bespoke prompt).
- **Falsifiable child-sid detection**: snapshot pre-existing sids in the project dir BEFORE
  `/clear`; bind the FIRST new sid whose first-record cwd matches the pane and whose ts
  post-dates the clear; on 0 or >=2 candidates, record NO lineage + toast (never guess).
- **Idle gate** (don't inject into a busy pane) + a trigger DISTINCT from b1's plain
  Shift+F11 (b1 = /compact; b2 = a deliberate variant, e.g. a modifier or a leader key).
